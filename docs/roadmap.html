A roadmap that reads like engineering work, not decorative ambition.

Priority bands

The roadmap is split by what actually unlocks the next class of project.

This page is intentionally structured around release leverage. Each band answers a different question: can teams author faster, can content scale, and can projects ship?

Authoring band

  • Prefab authoring and instance management.
  • UI editor completion and editor UX hardening.
  • Safer asset rename and dependency repair paths.

Content band

  • Animation state machines and skeletal playback.
  • Model import maturity for larger production scenes.
  • Better scene composition for reusable runtime content.

Shipping band

  • Standalone player build and packaging.
  • Particle, terrain, and broader runtime feature coverage.
  • More robust project lifecycle from prototype to release.

Architecture band

  • Keep render and tooling internals explicit as the engine grows.
  • Avoid accidental complexity that would erase the engine's main advantage.
  • Preserve scriptability while strengthening native ownership boundaries.
Immediate focus

What deserves attention right now.

These are the work items most likely to improve the engine's day-to-day usefulness for contributors and early projects.

Prefab lifecycle

Without prefabs, scene reuse and team collaboration stay heavier than they should. This is one of the highest leverage missing systems.

UI pipeline completion

The editor already exposes a strong base. Completing UI authoring turns that base into a more credible project workflow.

Asset GUID reliability

Rename and move operations must become safer if the engine is going to support larger repositories and less fragile collaboration.

Animation path

Animation is the bridge between a strong renderer and believable game content. It is the next major expansion in expressive scope.
