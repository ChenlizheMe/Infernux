Getting Started

InfEngine v0.1  |  中文

InfEngine is a C++17 / Vulkan game engine with a Python scripting layer. The C++ core handles rendering, physics, and resource management. Python handles gameplay logic, editor tooling, and render pipeline authoring. This guide takes you from a fresh clone to a running editor with your first scripted component.

Clone & Build

1. Clone with submodules

git clone --recurse-submodules https://github.com/ChenlizheMe/InfEngine.git
cd InfEngine

If you forgot --recurse-submodules:

git submodule update --init --recursive

2. Create a Python environment and install dependencies

python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt

3. Build the native module

cmake --preset release
cmake --build --preset release

This compiles the C++ engine, generates the pybind11 module, and installs the package so import InfEngine works immediately inside the venv.

Build time is roughly 5–10 minutes on a mid-range CPU on first build. Incremental builds are much faster.

Run the editor

python packaging/launcher.py

The launcher opens a project browser. Create or open a project folder to enter the editor.

Create a project

Click New Project in the launcher, choose a name and directory, then click Create. The launcher initialises a project folder with an Assets/ directory and a default scene.

Open a scene

After entering the editor, the default scene opens automatically. You can double-click any .scene file in the Project panel to switch scenes. To create a new scene, right-click in Project and choose Create → Scene.

Add a game object

Right-click in the Hierarchy panel and choose Create Empty, or use the GameObject menu. You can also rename the object in the Inspector, position it with the translate gizmo in the Scene View, and attach components from the Inspector's Add Component button.

Write a component

All Python gameplay logic lives in components that inherit from InfComponent. Create a .py file anywhere inside your project's Assets/ folder:

from InfEngine.components import InfComponent, serialized_field
from InfEngine.math import Vector3

class Rotator(InfComponent):
    speed: float = serialized_field(default=90.0)  # degrees per second

    def update(self, delta_time: float):
        t = self.transform
        angles = t.euler_angles
        t.euler_angles = Vector3(angles.x, angles.y + self.speed * delta_time, angles.z)

The editor hot-reloads scripts automatically when you save. Attach Rotator to any object via Add Component in the Inspector.

Serialized fields

Any class-level annotation decorated with serialized_field() is saved to the scene and shown as an editable field in the Inspector:

class Enemy(InfComponent):
    health: int    = serialized_field(default=100)
    speed: float   = serialized_field(default=3.5)
    label: str     = serialized_field(default="Enemy")

Supported types include int, float, str, bool, Vector2, Vector3, Vector4, and lists of those types.

Input

from InfEngine.input import Input, KeyCode

class PlayerController(InfComponent):
    move_speed: float = serialized_field(default=5.0)

    def update(self, delta_time: float):
        dx = dz = 0.0
        if Input.get_key(KeyCode.W): dz += 1.0
        if Input.get_key(KeyCode.S): dz -= 1.0
        if Input.get_key(KeyCode.A): dx -= 1.0
        if Input.get_key(KeyCode.D): dx += 1.0

        t = self.transform
        fwd = t.forward
        right = t.right
        move = (fwd * dz + right * dx) * self.move_speed * delta_time
        t.position = t.position + move

Lifecycle methods

MethodWhen it runs
awake()Once, when the component is first created or the scene loads
start()Before the first update(), after all awake() calls
update(dt)Every frame while the component is enabled
late_update(dt)After all update() calls this frame
on_enable()When the component or its game object becomes active
on_disable()When the component or its game object becomes inactive
on_destroy()When the component is removed or the object is destroyed
